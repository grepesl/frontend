API

Aplication Programming Interface - jūsų užklausa iš aplikacijos (front-end) yra išsiunčiama į serverį (back-end) ir jeigu viskas tvarkoje - tuomet grąžinami kokie nors duomenys pagal jūsų užklausą; jeigu kažkas negerai - grąžinama kažkokia error žinutė.

Kas yra API - video (pas ji nuoroda yra).

JSON

JavaScriptNotation - duomenų formatas naudojamas requst'uose (užklausose).
JSON formatas nuo JS objekto formato skiriasi tuo, kad: raktiniai žodžiai ir string reikšmės privalo būti dvigubuose kabutėse IR po paskutinio elemento masyve arba paskutinės reikšmės objekte NEDĖTI kablelio.

Tam, kad paverti JSON reikšmę į JS arba atvirkščiai, reikia naudoti JSON metodus:

  • JSON.parse(JSON_Duomuo) - paverčia JSON_Duomuo į JS sintaksę.
  • JSON.stringify(JS_Duomuo) - paverčia JS_Duomuo į JSON sintaksę

Fetch

Funkcija/metodas, kurio pagalba mes galime siūsti užklausas į serverį ir gauti kažkokius atsakymus.

  1. Fetch'inant (siunčiant užklausą) nurodomas URL, galima nurodyti papildomas opcijas. Tarp tų opcijų yra metodai:
    • GET - gauti informacija (default)
    • POST - siūsti info (sukurti naują duomenį)
    • PUT - redaguoti info (pagal ID) (perrašo viską apart ID)
    • PATCH - redaguoti info (pagal ID) (pakoreguoja arba prideda)
    • DELETE - trinti info (pagal ID)
  2. Apdoroti gautą info/atsakymą - paversti į JS duomenis
  3. Naudoti gautą apdorotą info
  4. Pagauti (catch) error ir kažką su juo daryti
  5. Galiausiai (finally) dar kažką atlieki (nesvarbu buvo ar nebuvo error)
