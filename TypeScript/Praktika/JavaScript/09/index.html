Masyvų interaciniai metodai

Masyvų interaciniai metodai yra tokie metodai, kurie suka ciklą per masyvą, ima kiekvieną jo elementą (kurį galima užsivadinti kaip nori) ir su juo kiekvienos interacijos metu atlieka kažkokius veiksmus /sąlygas / kažką grąžina.

Syntax:
1) masyvas.forEach((element) => {}) 2) masyvas.forEach((element, interacija) => {... veiksmai ...}) 3) masyvas.forEach((element, interacioja, masyvas) => {... veiksmai ...})

ForEach metodas - suka ciklą per visą masyvą ir kiekvienos interacijos metu TIKTAI atlieka kažkokius veiksmus.
ForEach negali grąžinti kažkokios reikšmės, neturi return.
ForEach negali būti sustabdytas, neturi break.

Filter

Syntax:
1) masyvas.filter((element) => {... veiksmai sąlyga su grąžinimu...}) 2) masyvas.filter((element, interacija) => {... veiksmai sąlyga su grąžinimu...}) 3) masyvas.filter((element, interacioja, masyvas) => {... veiksmai sąlyga su grąžinimu...})

Filter metodas - suka ciklą per visą masyvą ir kiekvienos interacijos metu gali atlikti kažkokius veiksmus ir tuomet interacijos elementą praleistii pro filtrą jeigu sąlyga yra tenkinama arba ne
Kitaip pasakius - išfiltruoja masyvą ir grąžina naują masyvą su išfiltruotomis reikšmėmis.

Map

Syntax:
1) masyvas.map((element) => {... veiksmai | sąlyga | grąžinimas naujo pavidalo...}) 2) masyvas.map((element, interacija) => {... veiksmai | sąlyga | grąžinimas naujo pavidalo...}) 3) masyvas.map((element, interacioja, masyvas) => {... veiksmai | sąlyga | grąžinimas naujo pavidalo...})

Map metodas - suka ciklą per visą masyvą ir kiekvienos interacijos metu gali atlikti ka=kokius veiksmus gali pagal kažkokią sąlygą gražinti arba gražinti kažkaip mutuotą/pakeistą/redaguotą elementą
Kitaip pasakius - gaunamas vienoks masyvas, gražinamas kitoks masyvas.

Reduce

Syntax:
1) masyvas.reduce((accumulator, currentElement) => {... veiksmai gražinimas}) 2) masyvas.reduce((accumulator, currentElement, interacija) => {... veiksmai gražinimas}) 3) masyvas.reduce((accumulator, currentElement, interacioja, masyvas) => {... veiksmai gražinimas...}) 4) masyvas.reduce((accumulator, currentElement) => {... veiksmai gražinimas...}, initialValue) 5) masyvas.reduce((accumulator, currentElement, interacija) => {... veiksmai gražinimas...}, initialValue) 6) masyvas.reduce((accumulator, currentElement, interacioja, masyvas) => {... veiksmai gražinimas...}, initialValue)

Reduce metodas - suka ciklą per visą masyvą ir kiekvien interacijos metu keičia accumulatorių naudodamas currentElement ir grąžina pakeistą accumulatorių kitai interacijai. Viso metodo galia yra grąžinama viena reikšmė - accumulator.
Jeigu nėra nurodytas initialValue - pirmosios iteracijos metu, accumulator yra 0'tasis elementas, o currentElement yra 1'asis elementas.
O jeigu yra nurodytas initialValue - pirmosios iteracijos metu, accumulator yra initialValue, o currentElement yra 0'tasis

Tas pats kas reduce tik iš kito galo

Junginys flat(1) ir map() metodų.

Syntax:
1) masyvas.some|every((element) => {... veiksmai sąlyga grąžinimas...}) 2) masyvas.some|every((element, interacija) => {... veiksmai sąlyga grąžinimas...}) 3) masyvas.some|every((element, interacioja, masyvas) => {... veiksmai sąlyga grąžinimas...})

some metodas - suka ciklą per visą masyvą ir kiekvienos iteracijos metu tikrina ar elementas atitinka kažkokią sąlygą. jeigu bent vienas elementaas atitinka kažkokią sąlygą, tuomet yra stabdomaas ciklas ir gržinama tiesa.

every metodas - suka ciklą per visą masyvą ir kiekvienos iteracijos metu tikrina ar elementas atitinka kažkokią sąlygą. jeigu bent vienas elementaas NEatitinka kažkokią sąlygą, tuomet yra stabdomaas ciklas ir gržinama false.

Syntax:
1) masyvas.find((element) => {... veiksmai sąlyga grąžinimas...}) 2) masyvas.find((element, interacija) => {... veiksmai sąlyga grąžinimas...}) 3) masyvas.find((element, interacioja, masyvas) => {... veiksmai sąlyga grąžinimas...})

Find - grąžina pirmąjį rastą elementą, kuris atitinka kažkokią sąlygą

FindIndex - grąžiną pirmojo rasto elemento indeksą, kuris taitinka kažkokią sąlygą

FindLast - tas pats kas Find, tik iš kito galo

findLastIndex - tas pats kas FindIndex, tik iš kito galo

Destruktūrizavimas - kažko (parametrai, objektai, masyvai, ?...) išskaidymas į atskiras dalis ir priskyrimas

  • Išrašant kintamuosius eilės tvaka, jiems priskiriamos reikšmės iš masyvo eilės tvarka
  • Galima priskiriant suteikti default/numatysasias rerikšmes. Jeigu destruktūrizuojamo masyvo reikšmė yra undefind - priskiriama nurodyta default reikšmė
  • Galima praleisti reikšmę, įrašant kablelį/nerašant naujo pavadinimo
  • Galima surinkti visas likusias reikšmes į vieną parašant daugtaškį (...) ir pavadinimą. Spread operatorius
  • Kreipiamės į raktinius žodžius tais pačiais pavadinimais.
  • Galima keisti raktinio žodžio pavadinimą rašant originalųjį raktinį žodžį -> dvitaškis (:) -> naujas žodis
  • Galima priskirti default reikšmes, jeigu destruktūrizuojamo objekto reikšmė būtų undefind arba net nebūtų tokio raktinio žodžio
  • galima iš objekto viduje esančio objekto išgauti reikšmes raktT:{raktV}
  • galima iš objekto viduje esančio masyvo išgauti reikšmes raktT:{skaicius:pav}
  • Galima išgauti visas likusias naudojant daugtaškį (...) spread operatorius.
