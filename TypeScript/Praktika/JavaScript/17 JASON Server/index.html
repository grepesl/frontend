JSON Server

JSON server yra vienas iš NPM (Node Pacage Manager), kurio pagalba galime susikurti lokalų API serverį iš savo JSON failo. Šiame serveryje galėsime turėti savo susikurtus duomenis, kurie privalo būti JSON faile, JSON specifiniu formatu. Paleidus JSON serverį naudojant JSON failą su duomeninmis, galėsime į jį (JSON server) kreiptis su Fetch užklausomis ir nusiūsti/gauti/redaguot/trinti jame esančius duomenis.

Tam, kad galėtume naudotis JSON serveriu reikia:

  1. npm(jis įsirašo kartu su Node'u)
  2. Įsirašyti json-server globaliai (npm i json-server -g)
  3. Susikurti JSON failą su specifiniu duomenų formatu
  4. Pasileisti JSON failą su json-server komanda (json-server data.json)

Terminal

Terminal Commands

  • cd - change directory - skirta naviguoti tarp aplankalų
  • ls - list - išvardina failus/aplankalus esančius dabartinėje direktorijoje(linux/mac)
  • dir - directories - švardina failus/aplankalus esančius dabartinėje direktorijoje(windows)
  • clear - išvalo k0omandinės eilutės vaizdą (not really)
  • extra stuff
    • TAB - užbaigia eilute arba pasiūlo galimus užbaigimus
    • CTRL C - atšaukia dabartinę koamdą/veiksmą/eilutę...
    • ARROW UP/DOWN - vaikšto per komandines eilutes istoriją

Terminal Navigation

  • / - sistemos/projekto pradžią
  • ./ - niekur nenukelia (lieki toje pačioje vietoje)
  • ../ - žingsnis atgal per vieną aplankalą
  • - - grįžti į buvusį aplankalą iš kurio tiką išėjai (tarsi cntr z)

JSON server Routes

*kiekvienas collection name gauna savo raute'ą

  • GET collectionName - gauname visus collectionName esančius duomenis
  • GET collectionName/id - gauname vieną specifinį duomenį iš collectionName pagal to duomens ID
  • POST collectionName - papildo collectionName esančius duomenis nauju duomenimi
  • PATCH collectionName/id - redaguoja collectionName esantį duomenį pagal ID
  • PUT collectionName/id - redaguoja collectionName esantį duomenį pagal ID
  • DELETE collectionName/id - trina collectionName santį duomenį pagal ID
