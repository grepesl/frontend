Moduliai

Moduliai - skirtingi/atskiri failai su kažkokiu savo kodu. Tas kodas (arba jo dalis) gali būti panaudotas kitų failų.

Svarbūs aspektai

  • Moduliuose gali būti įvairus kodas: duomenys (kintamieji), funkcijos, Klasės...
  • Norint pasiimti kažkokią informaciją iš kito failo, reikia ją import'uoti, o ta informacija turi būti export'uota
  • HTML'e prijungtas JS'as kuris kažką import'uoja - PRIVALO turėti atributą type su reikšme module (type="module")
  • HTML faile, prie kurio prijungtas modulis - PRIVALO būti pasileisti kaip serveris, kitu atveju modulis neveiks.
  • Klasės turi būti atskiri failai. Visiems failas - viena Klasė. Tiek Klasės, tiek Failo vardai turi būti vienodi ir iš didžiųjų raidžių
  • ...

Export ir Import

Failuose esantis kodas gali būti iškeliamas (export'uojamas) ir galima kažkokį kodą iš kito failo įsikelti (import'uoti).

Export

Duomenys, funkcjos, Klasės gali būti exportuojamos kaip paprasti dalykai ara kaip pagrindinis failo dalykas.
Iš vieno failo galima iškelti vieną pagrindinį (default) dalyką ir neapribotą kiekį papildomų dalykų.

  • Toje pačioje eilutėje (negalima arrow funkcijos kaip default)
  • Kodo apačioje išvardinti visus exportus {} ir/arba vieną export default

Import

iMPORTUOTI GALIMA DEFAULT ARBA PAPRASTUS EXPORTUS. sKIRTUMAS TAS, KAD DEFAULT NEREIKIA SKLIAUSTELIŲ IR NORINT PERVADINTI TIESIOG RAŠAI KITĄ PAVADINIMĄ BO JOKIŲ PAPILDOMŲ ŽODŽIŲ, O NORINT PERVADINTI PAPRASTUS REIKIA NAUOTI ŽODELI "as".

  • import { something as diff, somethingElse} from 'somewhere'
  • import defaultExportThingAnyName from 'somewhere'
  • import mainThing { something as diff, somethingElse } from 'somewhere'

Komponentai

Komponentas - HTML elementas sukurtas JS'u.

Komponentai dažniausiai yra kuriami framework'ų pagalba, tokių kaip: React/Vue/Angular/Swift... Bet juos galima kurti ir JS'e kaip atskiras Klases ar funkcijas.
