Libraries and Frameworks

Libraries - kažkieno kito parašytas kodas (kuris nėra originalioje kalboje), kuriuo galite naudotis savo projektuose.
Norint naudotis library kodu, reikia tą library prisijungti prie savo projekto panaudojant CDN.
Daugelis library veikia tokiu principu, kad jų viduje yra aprašytas kodas, kuriuo tu pats naudojiesi.
CSS libraries gali būti ClassLess, kurios stilius suteikinėja tagams.

frameworks - kažkieno kito sukurta programa/programū rašymo būdas, kuriuo naudojantis patogiau kurti programas.

Pros

  • Pagreitina/palengvina darbą
  • Dirbant k0omandoje vientisas tvarkingas kodas
  • Nereikia/labai mažai rašyti CSS
  • Įdiegtas responsivness

Cons

  • Reikia mokintis naudotis biblioteka
  • Neunikalus dizainas
  • Sudėtingiau pritaikyti savo stilius (dėl CSS prioritetų)
  • Daug perteklinio/nenaudoto kodo (užkrauna programą)
  • Nešvarus HTML dėl klasių pertėkliaus
  • Pasimirša CSS

Preprocessors

CSS with superpowers

Suteikia galimybę rašyti CSS kodą najiviškesniu būdu. Jums lengvai ir patogiai galima kurti kintamuosius, nestinti CSS kodą, rašyti: sąlygas, funkcijas, ciklus, mixinus ir t.t.

Naudojant betkurį CSS preprocesorių, reikia parašyti kodą konvertuoti (kompiluoti) į paparastą CSSą kurį jau prijungiame prie svetainės.

Units

Units - matavimo vienetai, skirti nustatyti elementų dydžius: plotį, aukštį, teksto dydį, tarpus, atitraukimus...

  • px - pikselis
  • % - procentas pagal tėvinio elemento dydį
  • vw - viewport width - matomos svetainės plotis (100 - visas plotis)
  • vh - viewport height - matomos svetaibės aukštis (100 - visas aukštis)
  • rem - root emm(ephemeral) - 16px(default) - pagal šakninis elementas
  • em - ephemeral - 16px(default) - pagal tėvinio elemento nustatymą
  • ...
